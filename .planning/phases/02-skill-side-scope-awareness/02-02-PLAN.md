    **Current behavior:**
    - Checks `grep -qF "// discord-local-scoping patch applied" "$SERVER_TS"` — if found, exits early
    - If not found, runs `patch -p0 < "$PATCH_FILE"`

    **Required change:**
    The script must now check BOTH the server.ts marker AND the SKILL.md marker. Only skip if BOTH are already applied. If either is missing, run the full patch (since `patch -p0` is idempotent for already-applied hunks — they'll be reported as "already applied" and skipped).

    Add these lines/variables:

    ```bash
    SKILL_MD="$HOME/.claude/plugins/cache/claude-plugins-official/discord/0.0.1/skills/access/SKILL.md"
    SKILL_MARKER="## Scope resolution"
    ```

    Change the already-applied check from:
    ```bash
    if grep -qF "$MARKER" "$SERVER_TS" 2>/dev/null; then
      echo "discord-channel: patch already applied — skipping" >&2
      exit 0
    fi
    ```

    To:
    ```bash
    server_patched=false
    skill_patched=false
    grep -qF "$MARKER" "$SERVER_TS" 2>/dev/null && server_patched=true
    grep -qF "$SKILL_MARKER" "$SKILL_MD" 2>/dev/null && skill_patched=true

    if $server_patched && $skill_patched; then
      echo "discord-channel: patch already applied — skipping" >&2
      exit 0
    fi
    ```

    The rest of the script (patch file present check, plugin installed check, `patch -p0` execution) remains unchanged. When `patch -p0` is run, it applies both the server.ts hunks and the SKILL.md hunk in one pass — already-applied hunks are handled gracefully by the patch tool.
